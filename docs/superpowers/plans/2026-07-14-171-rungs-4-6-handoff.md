## The remaining rungs

### Rung 4 — `SigningThreshold` (closes #130)

Migrate `Tholder` out of `cesr::core::primitives` into `cesr::keri` as `SigningThreshold`, absorbing #130's lean weighted representation + `satisfied_by(indices)`.

- **Current surface:** `cesr/src/core/primitives/tholder.rs` — `pub enum Tholder { Simple(u64), Weighted(Vec<Vec<(u64,u64)>>) }`, with `satisfy(indices)` (line ~33) and `check_well_formed`/`ThresholdError`. Consumed by: events (`threshold`/`next_threshold` fields), builders (`validate_threshold`), the parser conversion layer, both writer backends (`tholder_to_json`, `write_tholder`), and keri-rs `KeyState`.
- **Keep it pure numbers** — wire encoding lives on the event as `ThresholdForm` (rung 3), NOT inside the threshold value, so equality stays arithmetic. Do not fold form into `SigningThreshold`.
- **Breaking:** `Tholder` moves module + likely renames; `SerderError::SigningThresholdOutOfRange { field, source: ThresholdError }` and the builder `.threshold(...)` setters follow it. Call out in the PR + CHANGELOG.
- **Gate anchor:** the `keripy_parity::formulas` sweep (Tholder.satisfy vs keripy) + byte-identity corpora must stay green.

### Rung 5 — promote `DirectJson`, delete the JSON-tree writer (the core goal)

This is the "kill the python-in-JSON writer" rung. `DirectJson` (`cesr/src/serder/serialize/direct.rs`) is already a typed, buffer-direct canonical emitter, byte-identical to the `SerdeJson` reference backend under the cross-backend proptests. Promote it to the ONLY writer.

- **Delete from `cesr/src/serder/serialize.rs`** (verified present @ f1dc5e0): `trait EventSerializer` (~L175), `struct SerdeJson` (~L192), `fn serialize_with` (~L230), `enum AnchorJson` (~L461), `struct EventBody` (~L481), `fn seal_to_json` (~L519), `fn tholder_to_json` (~L573), `fn toad_json` (~L609), `fn matters_to_json_array` (~L632), and the splice helpers `fn abs_range` (~L346) / `fn find_subslice` (~L357) / `fn patch_slot` (~L366). The five per-ilk SerdeJson render paths in `serialize/{icp,rot,ixn,dip,drt}.rs` collapse into the DirectJson equivalents (direct.rs already has `write_tholder`/`write_toad`/seal/array writers).
- **The dummy-SAID-then-patch two-pass STAYS** — it's inherent to SAID computation — but DirectJson records slot spans by construction as it writes, so no buffer-searching (`find_subslice`) survives.
- **`serde_json` leaves the production write path.** It remains in the test-only tolerant reference reader (`deserialize/reference.rs`) and in `OpaqueSeal` handling on the read side (verbatim JSON text is the domain value there).
- **Replace the retired cross-backend proptest** (DirectJson vs SerdeJson) with **write→read→write fixpoint** properties against the strict reader over the builder-reachable event space (`event_strategies.rs` already supplies the strategies). Losing the two-backend differential is fine ONCE it's the only backend — but the fixpoint property must land in the same PR so coverage never dips.
- Biggest-diff rung; expect broad but mechanical changes. Byte-identity corpora are the safety net.

### Rung 6 — zero-copy `KeriEvent` (closes #129)

Borrow-ify `KeriEvent` across serder so parsing doesn't own-allocate the whole event. Genuine design work — brainstorm the borrow lifetimes against the current owned reified types (`cesr/src/keri/event/*.rs`) before planning. Composes with the now-clean single writer from rung 5.

## Load-bearing context carried from rungs 1–3 (do not re-derive)

- **keripy is the oracle, pinned** at `scripts/KERIPY_PIN` (`de59bc7d`). Local checkout `~/Code/keripy` with venv `~/Code/keripy/.venv` (Python 3.14). keripy's pysodium dlopens libsodium — on macOS prefix any keripy run with `DYLD_LIBRARY_PATH="$(nix build --no-link --print-out-paths nixpkgs#libsodium)/lib"`. If the venv is dead after the OS update (nix GC'd its Python), rebuild: `nix build --no-link --print-out-paths nixpkgs#python314`, then `python3 -m venv ~/Code/keripy/.venv && ~/Code/keripy/.venv/bin/pip install -e ~/Code/keripy`.
- **Byte-identity is law.** Rungs 4–6 must not change any wire byte. The #145 corpora gate this: `cesr/src/keripy_parity/events.rs` (26-vector sweep, all `reserialize:"identical"`, asserts count 26), `said_codes`, `seal_events`, and the two keri-rs fold differentials (`keri/tests/differential.rs` over `keystate.jsonl` + `kels.jsonl`). Any diff there = STOP, it's a real regression.
- **Domain types live in `cesr::keri`, not serder.** serder is a codec over keri-domain types. Rung 4 continues this by moving `Tholder` in. `cesr::core` stays CESR-encoding-only.
- **`cesr/src/keri/` is no_std + alloc.** Use `core::`/`alloc::` imports only; mirror `toad.rs`/`sequence.rs`/`threshold_form.rs`. Verify with `cargo build -p cesr-rs --no-default-features --features "alloc,core,b64,keri"`.
- **Fail-loud over silent, keripy is a conformance ref not a ceiling.** Rung 3 chose to reject (`IntegerFormOverflow`, `MixedThresholdForms`) where keripy silently re-renders — this is the house aesthetic; keep it and record divergences in the ledger.
- **Never paste keripy qb64 into `.rs` test source** — it trips the `cesr-typos` gate (base64 fragments read as misspellings) and file-glob excludes can't save a line inside a source file. Build test events in-code via the builders (e.g. `.threshold_form(Integer)`); leave keripy byte-agreement to the corpus sweep. This bit rung 3; see the note in `_typos.toml`.
- **Error discipline:** `thiserror`, one variant per failure domain, `#[source]`/`#[from]` preserves chains, a keri-domain type returns its own keri-domain error and `SerderError` bridges via `#[from]`. Name error variants by KERI domain vocab (rung 3 renamed to `cut`/`add`, matching keripy `br`/`ba`), not generic terms.
- **god-level clippy** (`all`+`pedantic`+`nursery` deny + restriction suite): fix code to the lint, every `#[allow]` carries a `reason`, never relax lint config.
