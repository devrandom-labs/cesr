## [Unreleased]

### Added

- *(ci)* spine phase 7 — tripwire gates. Two new flake checks encode the spine phases' architectural decisions so they cannot silently regress: `cesr-version-owner` fails if version-string wire grammar tokens (`KERI10`, `ACDC10`, `PPPPVVKKKK`, `b"KERI"`/`b"ACDC"`/`b"JSON"`/`b"CBOR"`/`b"MGPK"`, `const VERSION_STRING_LEN`/`VS_LEN` redefinitions) appear in production source outside the grammar's single owner `cesr/src/core/version.rs` (comment lines and `#[cfg(test)]`-gated items are exempt); `cesr-fn-ratchet` fails if any module's free `pub fn` count exceeds its recorded budget — the baseline (b64 6, core 0, crypto 6, keri 1, serder 70, stream 35, keri-rs 0) and the exact counting rule live in the new `free-fn-budget.toml`, and budgets may only be lowered. No library code changed.

- *(stream)* spine phase 4 — public group construction for the write spine. `ControllerIdxSigs::from_sigers(&[Siger<'_>]) -> Result<Self, ParseError>` and `WitnessIdxSigs::from_sigers(...)` build the `-A`/`-B` indexed-signature groups from a consumer's own signatures — the first public constructors on the attachment groups (all 27 stay otherwise crate-private). The count is derived from the input, never caller-supplied, so the count/raw invariant is held by construction; an empty slice yields a grammar-valid count-0 group (keripy's counter accepts count 0, `counting.py:878`; its `messagize` merely never emits one, `eventing.py:1605`). Both types are now also re-exported at `cesr::stream::{ControllerIdxSigs, WitnessIdxSigs}`. Additive.
- *(serder)* spine phase 4 — the write spine is complete. `SerializedEvent::frame_v1(&self, sigs: &ControllerIdxSigs, wigs: Option<&WitnessIdxSigs>) -> Result<Vec<u8>, FrameError>` frames a built event with its attachments as a KERI/CESR V1 message: body + `-V` attachment-group counter (count in quadlets, auto-promoting to `--V` above 4095 like keripy's `Counter`, `counting.py:872-875`) + `-A` controller sigs + optional `-B` witness receipts — the byte-exact mirror of keripy's `messagize` (`eventing.py:1605-1694`) and of `EventMessage::parse`, which round-trips its output. New `serder::FrameError` (thiserror: `MissingAuthenticator` mirroring messagize's refusal at `eventing.py:1582-1583`, `Encode(ParseError)` for counter-capacity overflows). Gated by `framed_inception_is_byte_identical_to_keripy` (`cesr/tests/spine_write.rs`): the fixture inception rebuilt from keripy's salt-derived seeds, re-signed in Rust, and framed — asserted byte-identical to keripy's `messagize` output. **Spec deviation, deliberate:** the spine spec (§2.3) names a free function `frame_event_v1`; it landed as the associated method `SerializedEvent::frame_v1` per the crate's behavior-lives-on-types convention (same call as phase 2's `EventMessage::parse`). Additive.

- *(serder)* spine phase 2 — the read spine. New `serder::message` module with `EventMessage<'a>`: `EventMessage::parse(&[u8]) -> Result<(EventMessage<'_>, &[u8]), EventMessageError>` parses one framed key event message (body + CESR V1 attachments, both keripy's framed `-V` layout and bare top-level groups) from the head of a stream and returns the exact remainder, so concatenated streams parse in a loop. The message carries the parsed `KeriEvent`, the exact signed byte span (borrowed from the input), and the attached controller (`-A`) / witness (`-B`) indexed signatures. New `serder::EventMessageError` (thiserror union — the first error type spanning the stream/serder seam): `Frame(ParseError)`, `Body(SerderError)`, `BareAttachment`, `UnexpectedGroup { group }`. **Spec deviation, deliberate:** the spine spec (§2.1) names a free function `parse_event_message`; it landed as the associated function `EventMessage::parse` per the crate's behavior-lives-on-types convention. Additive.
- *(keri-rs)* spine phase 2 — new opt-in `wire` feature (enables `cesr/serder`) with `Signed: From<&EventMessage>`: parsed wire messages feed `KeyState::incept`/`ingest` directly, and the `signed_bytes` provenance contract is held by construction. Default features unchanged — the sans-io #128 boundary stays the default. Covered by keripy-signed wire fixtures (`keri/tests/spine.rs`, generated by `scripts/keripy_spine_gen.py`). Additive.

### Fixed

- *(stream)* `encode_counter_v1` / `encode_counter_v2` now enforce their documented contract: a count that does not fit the counter code's soft field returns `ParseError::Malformed` instead of silently emitting an over-long (corrupt) counter (keripy rejects the same shape with `InvalidVarIndexError`, `counting.py:878-880`). Reachable through `encode_counter_auto_v1`/`_v2` only for codes without a big variant (e.g. V1 `-A` above 4095) or counts above the big capacity (`64^5 - 1`).

### Changed

- *(serder)* spine phase 6 — builder dedup, internal refactor. The five event builders move from the flat typestate (one struct carrying every field in every state plus a `PhantomData<State>` marker, with each transition re-listing all ~15 fields) to `core`'s per-state-struct style (`MatterBuilder`): each type state is a distinct struct carrying exactly the data accumulated so far, and transitions move whole values. The establishment payload the four establishment builders (`icp`, `dip`, `rot`, `drt`) accumulate identically is now shared scaffolding named by domain: `KeyConfiguration` (the `k`/`kt`/`n`/`nt` accumulator, in the new `builder::establishment` module) whose `validate()` is the single copy of the fourfold `build()` validation prologue (empty-keys, keripy majority defaults, integer-form fit, threshold well-formedness), returning the `SigningAuthority` the event asserts; `WitnessConfiguration` (`b`/`bt` for the inception family) and `WitnessRotation` (`br`/`ba`/`bt` against the prior set, for the rotation family) live in `builder::witness`, which stays the single home for witness validation. Public API preserved: every builder name, setter name/signature (including `const`-ness), and `build()` return type is unchanged, and all builder tests pass unmodified. Additive surface: new public marker trait `serder::builder::EventBuilderState` bounding the builders' state parameter (mirror of `core`'s `MatterBuilderState`); `InceptionBuilder::keys` / `DelegatedInceptionBuilder::keys` are now `const fn`. Behavior notes, all in dead or unpinned corners:
  - Dead defensive code removed: `rot`/`drt`/`ixn`/`dip` no longer store their type-state-required fields (`prefix`, `prior_event_said`, `delegator`) as `Option` and re-check them at `build()` — the checks were unreachable by construction. `SerderError::MissingBuilderField` is therefore never produced anymore; the variant is kept (documented as such) to avoid a breaking enum change.
  - Error-surfacing order in `rot`/`drt` `build()`: the `sn == 0` check now runs before the shared key-configuration validation instead of between its empty-keys check and its threshold checks. Observable only when `keys` is empty **and** `sn == 0` simultaneously — `SnBelowMinimum` now surfaces instead of `EmptyKeys`. Single-fault behavior (everything the tests and the keripy validation-parity suite pin) is byte-identical.
  - `sn` is stored as `u128` defaulting to 1 (was `Option<u128>` resolved at build); `witness_threshold` stays `Option<u32>` internally because `None` means "derive keripy's ample default", which needs the witness count only known at `build()`. No validation moved from `build()` to a setter, so no error surfaces earlier than before.
- *(stream)* [**breaking**] spine phase 5 — group unification. The ~20 hand-copied `{ raw, count }` group structs, their 18 one-function parse files, and the per-type `iter`/`into_vec`/`count`/`raw_bytes` impls are replaced by **one generic carrier per counting regime**, mirroring `core`'s `Matter<'a, C>` design: `stream::group::Group<K: GroupKind>` for element-counted groups and `stream::group::Frame<K: FrameKind>` for quadlet-counted framing groups. Each family is a sealed uninhabited kind in the new `stream::group::kinds` module (`ControllerIdxSig`, `SealSourceTriple`, `Attachment`, …) declaring its counter codes and element grammar (`element` for typed iteration, `skip` for lenient framing — the exact skip sequences of the old parsers, byte-for-byte); the two nested-counter families (`TransIdxSigGroup`, `TransLastIdxSigGroup`) carry custom grammars with the same validation messages. **All public group type names survive as aliases** (`ControllerIdxSigs = Group<ControllerIdxSig>`, `AttachmentGroup = Frame<Attachment>`, …), and `CesrGroup`, `Groups`/`GroupsV2`, `parse_group`/`parse_group_v2`, `count()`/`raw_bytes()`/`iter()`/`into_vec()`, and `ControllerIdxSigs::from_sigers`/`WitnessIdxSigs::from_sigers` are unchanged. Breaking details:
  - The 16 free encode wrappers are **deleted**: `encode_controller_idx_sigs_v1`, `encode_witness_idx_sigs_v1`, `encode_non_trans_receipt_couples_v1`, `encode_trans_receipt_quadruples_v1`, `encode_first_seen_replay_couples_v1`, `encode_seal_source_couples_v1`, `encode_seal_source_triples_v1`, `encode_trans_idx_sig_groups_v1`, `encode_trans_last_idx_sig_groups_v1`, `encode_attachment_group_v1`, `encode_generic_group_v1`, `encode_body_with_attachment_group_v1`, `encode_non_native_body_group_v1`, `encode_essr_payload_group_v1`, `encode_group_v1`, `encode_group_v2` (and the `impl_encode_element!`/`impl_encode_quadlet!` macros behind them). Encode through the `CesrEncode<V1>`/`CesrEncode<V2>` trait — four blanket impls on the carriers replace 50 macro-generated ones, and version safety is now type-level for **every** group: V2-only kinds do not implement the new `V1GroupKind`/`V1FrameKind` refinement traits, so encoding them as V1 does not compile (guaranteed by a `compile_fail` doctest on `V1GroupKind`).
  - The `stream::group::types` and `stream::group::iter` module paths are **removed**; every group name is importable from `stream::group` directly. `GroupIter<F>` (closure-parameterized) is replaced by the typed `stream::group::Elements<K>` iterator; `&Group<K>` now implements `IntoIterator`, and the old `GroupIter::{count, raw_bytes, into_remaining}` accessors are gone.
  - The 12 quadlet-counted newtypes (`AttachmentGroup`, `GenericGroup`, `PathedMaterialCouples`, …) no longer expose a public `.0` tuple field over `QuadletGroup`; use `quadlet_count()`, `raw_bytes()`, `to_bytes()`, and consuming `IntoIterator` on the `Frame<K>` alias instead.
  - No wire behavior changed: dispatch (the only place counter codes meet kinds), skip/element sequences, error messages, and counter capacities are ported verbatim — the keripy differential suites and all group tests stay green, and offset accumulation is now checked arithmetic in one place (`Group::parse`) instead of bare `+=` in 18 files.
- *(serder)* [**breaking**] spine phase 3 — deserialize now rejects malformed signing thresholds. `deserialize_event` / `deserialize_inception` / `deserialize_rotation` (and the delegated variants) enforce the same well-formedness the builders enforce, through the same shared helper (`validate_threshold` → `SigningThreshold::check_well_formed`): `kt` must be well-formed for the key count, and `nt` for the next-key count when next keys are committed. A wire event with `kt` exceeding `len(k)` (or `kt` 0, an empty weighted clause list, a weighted arity above the key count, or `nt` vs `n` mismatch) now fails with `SerderError::SigningThresholdOutOfRange` instead of parsing successfully and only being caught by the keri-rs fold. **Deviation from the phase card, deliberate:** no new `SerderError` variant — the failure domain already has one (`SigningThresholdOutOfRange`, the variant the builders return for the identical rule), and one variant per failure domain is the error-design law. Breaking: previously-accepted wire bytes are now rejected.
- *(keri-rs)* [**breaking**] spine phase 3 — the fold verifies witness receipts; `Signed.wigs` is load-bearing for the first time. New `Witnessing` (exported) pairs the event's governing witness set with its TOAD: each receipt is verified over `signed_bytes` against the non-transferable witness prefix its index selects — the declared `b` set at inception, the post-cut/add resolved set for a rotation, the state's carried set for an interaction (keripy `Kever.valSigsWigsDel`, `eventing.py:2735-2799` at the pin) — with out-of-range/invalid receipts skipped and duplicates counted once. At least TOAD distinct witnesses must have a valid receipt, else the new `Rejection::InsufficientWitnessReceipts { valid, required }` (terminal where keripy escrows partial witnessing; the consumer re-drives, same pattern as `OutOfOrder`). TOAD 0 is vacuously satisfied — unwitnessed KELs are unchanged. Covered by live-keypair receipt tests and a keripy-witnessed wire fixture (`keripy_icp_witnessed.cesr`, `scripts/keripy_spine_gen.py`).
- *(core,stream,serder)* [**breaking**] spine rung 1 — one owner for all version knowledge: `core::version`. `CesrVersion` is unified into a single enum (`V1`/`V2`, default `V2`) in `core::version` (no longer stream-gated), replacing both the dead `V1_0`/`V2_0` form and the duplicate in `stream::unwrap`; the `stream::CesrVersion` re-export and the crate-root `StreamCesrVersion` alias are **removed** — import `cesr::CesrVersion` or `cesr::core::version::CesrVersion`.
- *(serder)* [**breaking**] the `serder::version` module is **deleted**; `Protocol`, `SerializationKind`, and `VersionString` moved verbatim-in-spirit to `core::version` and are re-exported at `cesr::serder::{Protocol, SerializationKind, VersionString}`. `VersionString` now enforces its fixed-width invariants at construction (private fields; `new`/`with_size` return `Result` and reject `major`/`minor` > `0xF` and `size` > `0xFFFFFF`), exposes accessors (`proto()`, `major()`, `minor()`, `kind()`, `size()`), renders infallibly (`to_str() -> String`), and parses from bytes with a remainder: `parse(&[u8]) -> Result<(Self, &[u8]), VersionError>`. `Protocol::from_repr` / `SerializationKind::from_repr` now return `VersionError`.
- *(stream)* [**breaking**] `stream::parse_version_string` and `stream::parse_version_string_v2` are **removed**; use `core::version::VersionString::parse` / `VersionStringV2::parse`. `VersionStringV2` moved from `stream::message` to `core::version` as an owned, validated struct (`Protocol`/`SerializationKind` fields instead of `&str`/`ColdCode`; the protocol/genus majors are fixed at 2 by the frame and no longer stored) with `new`, accessors, `to_str()`, and remainder-returning `parse`. `stream::encode_version_string_v2` now takes `&core::version::VersionStringV2` and delegates to `VersionStringV2::to_str` — the shadow `kind_to_bytes` table in `stream::encode` is gone; `SerializationKind::as_str` is the only kind↔bytes table. Behavior note: the V1 parser now accepts hex major/minor digits (the old serder semantics, superset of keripy's `[0-9a-f]`) where the old stream parser accepted decimal only.
- *(core,stream,serder)* [**breaking**] new `core::version::VersionError` (thiserror, one variant per failure domain: `Truncated`, `UnknownProtocol`, `InvalidVersionDigit`, `UnknownKind`, `InvalidSizeHex`, `MissingTerminator`, `FieldOverflow`, `UnsupportedMajor`, `Base64`) is the single version-parse error. `stream::ParseError` gains a `Version(VersionError)` variant (`VersionError::Truncated` converts to `ParseError::NeedBytes`, preserving streaming retry semantics). `serder::SerderError` gains `Version(#[from] VersionError)` and **loses** `VersionStringOverflow` (now `VersionError::FieldOverflow` inside `Version`); `SerderError::InvalidVersionString` remains only for the serder-level checks (non-JSON kind on the strict read path, size field contradicting the input length).
- *(stream)* the sealed `stream::Version` trait gains `const VERSION: CesrVersion`, linking the `V1`/`V2` phantom marker types to their value-level `CesrVersion` tags.

- *(keri,serder)* [**breaking**] #129 zero-copy `KeriEvent<'a>` (rung 6 of #171, the final rung). `InceptionEvent`, `RotationEvent`, `InteractionEvent`, `DelegatedInceptionEvent`, `DelegatedRotationEvent`, `KeriEvent`, `Seal`, and `OpaqueSeal` gain a lifetime parameter — code naming them bare writes `<'static>` or `<'_>`; all gain `into_static()`. The events are covariant (compile-probed): **Vec lists were chosen over #129's original `Cow` lists** after a rustc variance probe showed `Cow`'s `ToOwned` projection makes the events invariant. `deserialize_event` / `deserialize_*` return events borrowing the input buffer; `KeriDeserialize` still returns owned events (near-free `into_static`). qb64 decode still allocates — the borrow covers `Matter` soft fields and opaque-seal payloads; the payoff is a future qb2 reader (the event shape is now ready). No wire bytes changed.
- *(serder)* [**breaking**] #171 rung 5 — single JSON writer, serde-free production path. Removed from the public API: `EventSerializer`, `SerdeJson`, `DirectJson`, `serialize_with`, `EventLayout` (now crate-internal), and the `SerderError::Json` error variant. Event serialization goes through the unchanged `serialize` / `serialize_*` / `KeriSerialize` entry points; the single canonical JSON writer produces byte-identical output (gated by the keripy byte-identity corpora). Renamed `serder::version::SerKind` → `SerializationKind`; it now carries the write-path dispatch (`render`), failing loud with the new `SerderError::UnsupportedSerializationKind` for kinds without a body codec (CBOR/MGPK/CESR), mirroring the strict reader. `serde`/`serde_json` are no longer production dependencies of the `serder` feature (dev-dependencies only). No wire bytes changed.
- *(keri)* [**breaking**] #130 migrate `Tholder` → `cesr::keri::SigningThreshold` — the signing-threshold domain type moves from `core::primitives` into `keri` (rung 4 of the #171 serder redesign). Weighted thresholds now use a flattened representation (`WeightedThreshold`: one contiguous weights buffer + cumulative clause boundaries), cutting per-threshold allocations from `1 + N` to `2`. `satisfy` → `satisfied_by`; `ThresholdError` → `SigningThresholdError` (adds `TooManyWeights` for the flattened-repr `u32` boundary overflow). No wire-format change — the #145 keripy byte-identity corpora stay green.

### Changed

- **core (#76):** `ValidationError` gains a new `SizeOverflow` variant, returned
  when computing a variable-size primitive's full size from its decoded soft field
  overflows `usize`. As `ValidationError` is public and not `#[non_exhaustive]`,
  this is **breaking** (MINOR under 0.x) for downstream exhaustive `match` on it.
- **error ergonomics (#33):** removed the `terrors::OneOf` error-union layer in
  favour of purpose-built `thiserror` enums. **Breaking** (MINOR under 0.x):
  - `MatterBuilder::{from_qualified_base64, from_qualified_base2, build}` now return
    `Result<_, MatterBuildError>` (variants `Parsing`, `Validation`) instead of
    `OneOf<(ParsingError, ValidationError)>`.
  - `crypto::verify` now returns `Result<(), VerificationError>` (variants
    `Signature`, `CodeMismatch`) instead of `OneOf<(SignatureError, CodeMismatchError)>`.
  - The indexer builder's parse/validation methods (`from_qb64`, `from_qb2`,
    `with_index`, `with_indices`, `with_raw`) return the bare `IndexerParseError` /
    `IndexerValidationError` (previously wrapped in a single-element `OneOf`).
  - Consumers matching on these results switch from `.take::` / `.narrow::` to a
    normal `match` on the new enums / bare types.
  - `SerderError` gains a new `UnparseablePrimitive` variant (see _Fixed_ below).
    As `SerderError` is public and not `#[non_exhaustive]`, this is **breaking** for
    downstream exhaustive `match` on it.
  - The `terrors` dependency is dropped.
- **core / serder (#67):** qb64 text encoding moved to `core`. **Breaking**
  (MINOR under 0.x):
  - `Matter<C>` gains infallible `to_qb64() -> String` and `to_qb64b() -> Vec<u8>`;
    encoding a key/digest/signature to text no longer requires the `stream` feature.
  - `stream::encode::matter_to_qb64` **removed** (use `Matter::to_qb64b`).
  - `serder::primitives::{to_qb64_string, identifier_to_qb64_string}` now return
    `String` instead of `Result<String, SerderError>`; the internal
    `serder::serialize::{seal_to_json, matters_to_json_array}` helpers likewise
    became infallible.
  - `SerderError` loses the `Qb64Encoding` and `Encoding` variants (their only
    producers are gone). Breaking for downstream exhaustive `match` on `SerderError`.

### Fixed

- **core (#76):** `MatterBuilder::{from_qualified_base64, from_qualified_base2}` no
  longer compute the frame size (`fs = size * 4 + cs`, `bfs = ceil(fs * 3 / 4)`) from
  the attacker-controlled soft field with unchecked arithmetic. The three sites now
  use `checked_mul`/`checked_add` and return `ValidationError::SizeOverflow` on
  overflow, per the arithmetic-safety rule — previously a large declared size would
  panic in debug or silently wrap in release (a truncated frame slicing as valid).
  Latent, not reachable through the parse API today (variable codes cap the soft
  field at `ss=4`, so `size <= 2^24-1`); fixed as a defense-in-depth rule violation.
- **serder (#33):** a malformed-but-unparseable field value no longer collapses a
  `ParsingError` into `ValidationError::UnknownMatterCode(..)` via string
  formatting; a new `SerderError::UnparseablePrimitive { field, source }` variant
  carries the parsing error in its own failure domain.
- **stream (#33):** removed an `unreachable!()` panic on the matter-parse error path
  in `stream::parse::parse_matter`; the error mapping is now a total `match`.
- **core (#33):** `MatterBuilder::from_qualified_base64` no longer panics
  (`range end index N out of range for slice of length 0`) on a malformed qb64 whose
  decoded buffer is shorter than the code's declared lead size (e.g. `5BAA`). The
  lead-byte slices are now bounds-checked and return
  `MatterBuildError::Validation(ValidationError::StructuralIntegrityError)`. Found by
  the `deep-fuzz` `matter_from_qb64` target; the crash input is committed as a fuzz
  corpus regression seed.

### Added

- **crypto/devx (#69):** indexed signatures (`Siger`, the form attached to KERI
  events) can now be verified directly — closing the sign/verify asymmetry where
  `sign_indexed` produced a `Siger` but `verify` only accepted a `Cigar`. This
  lands as a **type-unified** verify surface rather than a one-off method:
  - A `crypto::Signature` trait implemented by both `Cigar` (non-indexed) and
    `Siger` (indexed), so a single generic `verify` covers both — the caller
    never branches on "indexed or not".
  - `KeyPair::<A>::verify<S: Signature>(&self, data, &S) -> Result<(), SignatureError>`
    — one generic method (was three duplicated `verify` methods) with per-curve
    crypto dispatched on `A` at **compile time** via the new
    `Algorithm::verify_bytes`. `kp.verify(msg, &cigar)` and `kp.verify(msg, &siger)`
    both work.
  - Free `crypto::verify<S: Signature>(verfer, data, &S)` — the verifier-key-driven
    form (mirrors keripy's `siger.verfer.verify(siger.raw, ser)`) for verifying
    with only public keys. Composes into lazy iterator chains over `stream`-parsed
    signature groups: `sigers.try_for_each(|s| verify(verfer, msg, s))`.
  - Verification is **strict**: a signature whose CESR code does not belong to the
    key's algorithm is a typed error, not a silent failure. The `Siger` index is
    CESR framing metadata and is not part of the signed payload.
  - Also adds `Algorithm::owns_indexed` and `Algorithm::NAME`.
  ([#69](https://github.com/devrandom-labs/cesr/issues/69))

### Breaking

- **crypto (#69):** verification now returns `Result<(), _>` instead of
  `Result<bool, _>`. `Ok(())` means verified; a cryptographically invalid
  signature is the new `SignatureError::Invalid`, moved out of the success channel
  so `verify(..).is_ok()` can no longer mistake a forgery for a valid signature.
  Affects `KeyPair::verify` and the free `crypto::verify`. Callers change
  `if kp.verify(..)?` to `kp.verify(..)?;` (propagate) or match on the error.
- **crypto (#69):** new `SignatureError::Invalid` and
  `SignatureError::CodeMismatch { expected, actual }` variants; `SignatureError`
  is not `#[non_exhaustive]`, so exhaustive downstream `match`es must add arms.
  The `Algorithm` trait gains required items (`NAME`, `verify_bytes`) — but it is
  **sealed**, so no external impls are affected.

  ~equal to `main` at N≥16 groups). Borrowed `Matter<'a>` and a parser-combinator
